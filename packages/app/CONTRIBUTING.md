# Contributing to @rulesnap/app

## Development Setup

```bash
# From the monorepo root
pnpm install

# From packages/app/
pnpm start    # Dev server with platform picker
pnpm web      # Web dev server
pnpm test     # Run tests
```

## Adding a New Game

Create three things — the registry, routing, home screen, and SideNav update automatically.

### 1. Game config — `games/{game-id}/config.ts`

```typescript
import { GameConfig } from '@/types/GameConfig';

export const config: GameConfig = {
    id: 'my-game',
    sections: ['overview', 'setup', 'gameplay'],
    headerColor: '#2a4a3a',
    bggUrl: 'https://boardgamegeek.com/boardgame/12345/my-game',
    manualUrl: 'https://rulesnap.com/manuals/my-game.pdf', // optional
    sectionComponents: {
        overview: () => import('./sections/Overview').then(m => ({ default: m.Overview })),
        setup: () => import('./sections/Setup').then(m => ({ default: m.Setup })),
        gameplay: () => import('./sections/Gameplay').then(m => ({ default: m.Gameplay })),
    },
};
```

### 2. Section components — `games/{game-id}/sections/*.tsx`

Each section is a named export:

```typescript
export function Overview() {
  return ( /* section content */ );
}
```

### 3. Translations — `assets/i18n/{game-id}/en.json`

Must include `app.title` and `app.icon`. Add a `{section}.sidenav` key for each section:

```json
{
    "app": { "title": "My Game", "subtitle": "Quick Reference", "icon": "🎮" },
    "overview": { "sidenav": "Overview" },
    "setup": { "sidenav": "Setup" }
}
```

Then add imports to `games/translations.ts`:

```typescript
import myGameEn from '../assets/i18n/my-game/en.json';

export const translationMap = {
    // ...existing
    'my-game': { en: myGameEn },
};
```

### 4. Theme (optional) — `games/{game-id}/theme.ts`

```typescript
import type { Theme } from '../../theme/themes';

export const theme: Theme = {
    colors: {
        background: '#1a2a1a',
        card: '#2a3a2a',
        text: '#e0e0e0',
        textMuted: '#a0b0a0',
        accent: '#4caf50',
    },
    fonts: { body: 'System', heading: 'System' },
    spacing: { sectionMargin: 24, cardPadding: 16, containerPadding: 16 },
};
```

Without a theme, the game uses the home theme.

## Auto-Generated Files

These files are generated by scripts and should not be edited manually:

- `games/registry.ts` — generated from `games/*/config.ts`
- `theme/themeMap.ts` — generated from `games/*/theme.ts`

Both are gitignored and regenerate on `pnpm start`, `pnpm build`, `pnpm test`, or on file save during dev (via chokidar in Metro).

To add a new generator, drop a `generate-*.js` file in `scripts/` that exports a `generate()` function. It will be picked up automatically.

## Testing

```bash
pnpm test
```

- Jest with ts-jest
- Property-based tests use `fast-check` with `{ numRuns: 100 }`
- Test files go in `__tests__/`

## Linting & Formatting

```bash
pnpm lint:check    # Check for lint errors
pnpm lint:fix      # Auto-fix lint errors
pnpm format        # Format with Prettier
```

## Editing Existing Games

- **Sections** — Edit `games/{game-id}/sections/`. Hot-reloads.
- **Config** — Edit `games/{game-id}/config.ts`. Registry regenerates automatically.
- **Theme** — Edit `games/{game-id}/theme.ts`. Theme map regenerates automatically.
- **Translations** — Edit `assets/i18n/{game-id}/*.json`. Picked up on reload.

## Adding a New Locale

1. Create `{locale}.json` in each `assets/i18n/{game-id}/` and `assets/i18n/common/`
2. Add locale imports to `games/translations.ts`
3. Add the locale to `SUPPORTED_LOCALES` and `LOCALE_META` in `components/SideNav.tsx`
